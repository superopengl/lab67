# Deploy

AWS CDK app that provisions everything needed to host KidPlayAI in `ap-southeast-2`: VPC, Aurora Postgres Serverless v2, EFS (sandbox persistence), ECR, Fargate service, ALB, ACM cert, Route53 alias, and Secrets Manager entries.

## Architecture

```
Route53 alias (kidplayai.techseeding.com.au)
  └─ ALB (HTTPS, ACM cert via DNS-validation)
       └─ Fargate Service (1 task, 1 vCPU / 2 GB, public subnet + public IP)
            ├─ Container: techseeding/kidplayai (port 80; SG only allows ALB)
            ├─ EFS mount: /var/kidplayai  ← sandbox folders (TMPDIR override)
            ├─ Secrets:   PG_*, KPAI_JWT_SECRET, KPAI_SANDBOX_DEEPSEEK_API_KEY
            └─ Logs:      /kidplayai/<stage>
  └─ Aurora Postgres 16.6 — Serverless v2 (0.5–2 ACU), isolated subnets
```

The `/var/kidplayai` EFS mount + `TMPDIR=/var/kidplayai` make `os.tmpdir()` resolve to EFS, so sandbox folders persist across container restarts.

DB credentials are auto-generated by Aurora into Secrets Manager. The container reads `PG_HOST`, `PG_PORT`, `PG_USER`, `PG_PASSWORD`, `PG_DATABASE` from that secret and `docker-entrypoint.sh` composes `KPAI_DATABASE_URL` from them.

The Route53 hosted zone for `techseeding.com.au` is auto-resolved via `HostedZone.fromLookup`. The lookup result is cached in `cdk.context.json` (committed) so subsequent synths don't need Route53 list permissions.

## Prereqs

- AWS account + credentials (`aws sts get-caller-identity` should work)
- Node 24, pnpm 10
- Docker with buildx (image is built `linux/amd64`)
- One-time per account/region: `pnpm bootstrap`

## First deploy

```bash
cd deploy
pnpm install
pnpm bootstrap                          # one-time CDK bootstrap in ap-southeast-2
STAGE=prod ./scripts/deploy.sh          # provision infra + build image + push + redeploy service
```

Then populate the DeepSeek key (CDK creates an empty placeholder secret):

```bash
aws secretsmanager put-secret-value \
  --secret-id kidplayai/prod/deepseek \
  --secret-string 'sk-...' \
  --region ap-southeast-2
```

After the first deploy completes, browse to `https://kidplayai.techseeding.com.au`.

## Iterating on the image

`./scripts/deploy.sh` runs everything end-to-end. To skip the CDK step (no infra changes), just push a new image and force a redeploy:

```bash
TAG=$(git rev-parse --short HEAD) ./scripts/build-and-push.sh
aws ecs update-service \
  --cluster $(aws cloudformation describe-stacks --stack-name KidPlayAi-prod --query "Stacks[0].Outputs[?OutputKey=='ClusterName'].OutputValue" --output text) \
  --service $(aws cloudformation describe-stacks --stack-name KidPlayAi-prod --query "Stacks[0].Outputs[?OutputKey=='ServiceName'].OutputValue" --output text) \
  --force-new-deployment
```

## Common commands

From the repo root:

```bash
pnpm -F @techseeding/kidplayai-deploy synth     # render CloudFormation
pnpm -F @techseeding/kidplayai-deploy diff      # diff against deployed stack
pnpm -F @techseeding/kidplayai-deploy deploy    # cdk deploy --all
pnpm -F @techseeding/kidplayai-deploy migrate   # run drizzle-kit migrate as a one-off ECS task
```

Tail logs:

```bash
aws logs tail /kidplayai/prod --follow --region ap-southeast-2
```

## Overriding the domain

Defaults are `kidplayai.techseeding.com.au` in the `techseeding.com.au` hosted zone. Override via context or env:

```bash
pnpm exec cdk deploy \
  -c domainName=staging.example.com \
  -c hostedZoneName=example.com
```

## CI/CD (optional)

`.github/workflows/deploy.yml` runs on push to `main` (or manual dispatch). It needs:

- `AWS_DEPLOY_ROLE_ARN` repo secret — IAM role trusting GitHub OIDC for this repo
- Repo variables (optional): `AWS_REGION`, `KPAI_DOMAIN_NAME`, `KPAI_HOSTED_ZONE_NAME`

If you're building locally and pushing manually, you can ignore this file or delete it.

## Notes / caveats

- **Single task by default.** WebSocket connections (terminal PTYs) are bound to whichever task accepted them. Scaling beyond 1 needs sticky sessions or external state. Bump `desiredCount` in `lib/kidPlayAiStack.js` once you've solved that.
- **EFS performance.** Bursting throughput is fine for kid-scale traffic. Switch to `ThroughputMode.ELASTIC` if you see I/O backpressure.
- **Cost floor.** Aurora Serverless v2 (0.5 ACU min) + Fargate 1 vCPU/2GB + ALB ≈ USD ~45–60/month idle. No NAT gateway — Fargate runs in public subnets with a public IP for outbound only; inbound is gated by the SG (only ALB can reach port 80).
- **Destroying.** `pnpm destroy` retains the ECR repo and (in prod) snapshots the DB. Drop `RemovalPolicy.RETAIN` in code before tearing down for good.
